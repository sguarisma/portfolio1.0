Context

I collaborated with a four-person team, where each of us owned major features of the project over a 4-week sprint. We used React, Firebase, Clerk, TypeScript + CSS, and Figma to design and build a responsive web app from scatch. As a full-stack developer, I worked on the user registration, messaging, and peer matching components for StudBud.

Problem

Students in large lecture-based college courses often struggle to form study groups. Despite wanting to collaborate, logistical and social barriers, like not knowing anyone in the class or feeling awkward reaching out, makes it difficult to connect with peers. Based on both scholarly research and user interviews with students at Brown, we found that

  • Students with stronger academic peer networks experienced higher levels of motivation and learning
  • Students who did not participate in study groups most commonly cited logistical difficulties, like not being able to find a group, as the cause.
  • Access to collaborative study groups would save significant time, energy, and stress.
While tutoring apps and Canvas discussions exist, none offer an intutiive, class-specific social experience. Studbud was designed to fill this gap, matching students based on shared courses and enabling meaningful peer interaction.

Research

To begin, we combined quantitiative data, interviews with 25 Brown students, and competitive analysis to inform our product design. We found that students wanted to find study partners based on their current courses, a profile section with bios and photos increases trust and approachability, and messaging needed to be simple, low-pressure, and available asynchronously.

User personas we built showed a strong preference for convenience, privacy, and the ability to revisit matches and message before big exams. This led us to prioritize six core features:

  1. University email authentication
  2. Editable user profiles
  3. Course input & validation
  4. Course-based peer matching
  5. Direct messaging with canned starter phrases
  6. Secure data storage

Process

We followed a modular design approach and split development by feature area. My main responsibility was the Authentication module, Messaging module, and Matching module including:

  1. Clerk authentication with @brown.edu email and SSO

  2. Cross-User Messaging Functionality

  3. Peer Course Matching

This process included both frontend development using React and Clerk session management and data storage using Firebase Firestore for persistance across user sessions. I also helped integrate API requests to the Courses@Brown API, an API containing up-to-date information on all of Brown University's courses.

Solution

Overall, the completed StudBud prototype allows students to:

  • Register securly with a Brown email and SSO
  • Build a profile that reflects their interests and study preferences
  • Enter and standardize their current courses
  • View classmates by shared courses on a dynamic homepage

    • Match and message peers with intuitive chat tools and conversation starters

Our chat system supports unlimited messages (text, images, and emojis), asynchronous delivery (with timestamps), and persistent conversations with Firebase storage.

Learnings

This project was an essential learning opportunity in both full-stack development and team collaboration. This helped me develop across both technical and human-centered design dimensions. I leaned how to translate insights from user interviews into tangible features that directly addressed students' needs. I also gained hands-on experience managing and persisting user sessions and profile data using Firebase, while integrating Clerk for secure authentication.

In the future, this project could be built upon by adding components like:

  • Advanced AI matching to suggest study partners based on profile information
  • Greater integration with school systems so that students could automatically link their registered courses without the need to manually input course information
  • Calendar integration to easily plan study sessions within the app

One of the most valuable takeaways was learning how to work modularly in a collaborative team, balancing independent ownership of features with regular integration and feedback cycles. I became more confident writing scalable React components, working with real-time databases, and navigating version control in a group setting. Overall, StudBud was my first full developed full-stack project, one that taught me not just to build software, but how to center users in the process and collaborate efficiently to bring an idea to life.
