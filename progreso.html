Progreso Latino


Redesigning the contact page of a local non-profit's website to improve accessibility, usability, and mobile responsiveness.

Context

I worked independently on this case study to redesign the Contact Us page of Progreso Latino, a nonprofit that serves Latine and immigrant communities in Rhode Island.

Problem

This project was an opportunity to practice the principles of responsive web design and accessibility. I chose Progreso Latino's website not only because their mission resonated with me, but also because their site, particularly their Contact page, was difficult to use across devices.

Research

To begin, I conducted an evaluation of the Contact page, considering various factors. I was able to identify several key issues with the original design:

  • Efficiency (LOW): The navigation bar spanned almost half the page with excessive buttons. The cluttered layout of the page overall slowed users down and distracted from key content. The misleading page URL also made it difficult to trust or locate information on the greater site (part of the url for the contact form's web page was gala-2018).
  • Learnability (MEDIUM): While the contact form offered descriptive feedback on input errors and it had an easy-to-find accessible/language settings button, the overall layout lacked organization. With the navigation bar, users had to explore and guess what content each link might contain, making it less intuitive.
  • Memorability (LOW): The design lacked visual consistency, particularly noticable from the different fonts and mismatched shades of blue that diminished brand cohesion. It also lacked visual hierachy, making it harder to scan the page. It did have a clear red and blue color brand and had familiar features, like the navigation bar.
  • Conceptual Model Alignment: While the contact form followed familiar patterns, the navigation bar posed a problem in its abnormal size. Additionally, the page mentions a balloon icon meant to link to Google Maps, but the icon doesn't appear anywhere on the page, disrupting user expectations.

To assess accessibility issues, I ran the webpage through WebAIM WAVE. This process flagged many issues, like missing form labels, empty links, unclear alt text, and inconsistent font sizes. This research confirmed that the site lacked accessibility and usability, particularly in the mobile versions, thus requiring a responsive redesign.

Solution

Finally, I built out the final redesign using HTML and CSS. The final site can be found here or by clicking the final redesign image below.

In this final build, I used Flexbox to align key elements into clean vertical or horizontal groupings. As the screen size shrinks, the layout shifts to a more vertical flow, optimizing readibility. Key updates of the redesign include:

  • A cleaner navigation bar, with a hamburger menu for mobile/tablet, to avoid a cluttered look and increase ease of touchscreen use. This would help increase the efficiency and learnability of the page as users can clearly navigate around the site intuitively.
  • Full-width form inputs and labels with proper tags that will be the page easier to use through touch, as well as more responsive and accessible overall.
  • Clearly differentiated hover and active states for components, increase accessibility of the site.
  • Consistent styling and layout by keeping with the colors, fonts, and button components of the style guide, ultimately helping with the page's memorability.
  • Fully responsive design, also increasing accessibility for users of various devices.

Learnings

This project taught me the importance of designing for flexibility, revealing how a single page, like a contact form, can have exclusionary UX issues. I learned to use flexbox as a tool for responsive layouts and overall how to engage with the design process intentionally and responsibly to guarantee a site that was both responsive and accessible. In the future, I would love to improve on my designs and possibly perform usability tests from real users, particularly with the navigation bar.
